Overview

lapakID adalah platform jual beli ID premium dengan sistem tier, pembayaran QRIS, dan dashboard admin lengkap. Dibangun dengan HTML/CSS/JS murni di frontend dan Node.js serverless di backend (Vercel), dengan database MongoDB Atlas.

Platform ini berjalan sepenuhnya di Vercel dengan fungsi serverless. Tidak memerlukan server terpisah — cukup deploy ke Vercel dan set environment variables.

Stack Teknologi

  • Frontend: HTML5, CSS3, JavaScript (Vanilla) + Lucide Icons
  • Backend: Node.js Serverless (Vercel Functions)
  • Database: MongoDB Atlas (cloud)
  • Payment: QRIS dinamis (CRC16 algorithm)
  • Deployment: Vercel (auto-deploy dari GitHub)

Fitur v2.2.2

  • Sistem Tier: Low, Medium, High, Legend dengan harga berbeda
  • Pagination: 20 ID per halaman per tier
  • Tag NEW & SOLD: Otomatis berdasarkan tanggal (3 hari untuk NEW)
  • Kode Promo: Diskon persentase dengan batas penggunaan & expiry
  • QRIS Dinamis: QR code dengan nominal otomatis (CRC16)
  • Live Chat: Chat real-time dengan admin, polling 8 detik
  • Notifikasi: Bell notifikasi untuk pembelian yang dikonfirmasi
  • Music Player: Playlist dari database, marquee untuk judul panjang
  • Banner & Popup: Dikelola dari dashboard admin
  • Dark Mode: Toggle tema dengan persistensi localStorage
  • Sound FX: Suara klik saat ID dipilih, suara sukses saat bayar
  • GitHub Editor: Edit file langsung dari dashboard, push otomatis

Sistem Tier

Setiap ID dikategorikan ke dalam 4 tier berdasarkan keunikan dan nilai nomor:

TierDeskripsiHarga Default
LOWID biasa, kurang spesialRp 125.000
MEDIUMID cukup unikRp 450.000
HIGHID sangat unik / cantikRp 850.000
LEGENDID super langkaRp 1.350.000
Harga dapat diubah dari Dashboard Admin → tab Pengaturan. Tier Legend mendukung harga khusus per ID.

API Reference

Semua endpoint berada di /api/*. Admin endpoint membutuhkan header x-admin-token.

IDs

GET/api/ids
Ambil semua ID. Query: ?tier=low&sold=false&search=xxx&sort=likes
GET/api/ids/popular
12 ID paling banyak disukai
GET/api/ids/stats
Statistik: total, terjual, tersedia, likes
POST/api/ids
Tambah ID baru (Admin). Body: {number, tier, note}
POST/api/ids/bulk
Upload massal (Admin). Body: {ids[], tier, note}
PUT/api/ids/:number
Update ID (Admin). Body: {tier?, sold?, note?, likes?}
DELETE/api/ids/:number
Hapus ID (Admin)

Like, Payment, Promo

POST/api/like/:number
Like ID (1x per IP, anti-spam)
POST/api/payment
Buat pesanan. Body: {idNumber, method, buyer, phone, promoCode?}
PUT/api/payments/:id/confirm
Konfirmasi pembayaran (Admin) — otomatis tandai ID terjual & buat notifikasi
POST/api/promo/validate
Validasi kode promo. Body: {code}

Settings, Notif, Reports

GET/api/settings
Semua settings: prices, adminFee, music, banners, popup
PUT/api/settings/:key
Update setting (Admin). Body: {value: any}
GET/api/notifications
Notifikasi untuk publik (confirmed + info)
POST/api/reports
Kirim pesan live chat (1x per IP per hari)
GET/api/reports/reply-check
Cek balasan admin. Query: ?name=xxx
PUT/api/reports/:id/reply
Balas pesan user (Admin). Body: {reply}

GitHub Editor

GET/api/github/file?path=index.html
Ambil konten file dari GitHub (Admin)
PUT/api/github/file
Push perubahan file ke GitHub (Admin). Body: {path, content, sha, message?}

Cara Beli

  1. Browse ID — Lihat daftar ID berdasarkan tier, atau gunakan fitur pencarian
  2. Klik ID — Buka modal detail untuk melihat harga dan info lengkap
  3. Klik "Beli ID Ini" — Diarahkan ke halaman payment
  4. Isi Data — Nama lengkap dan nomor WhatsApp aktif
  5. Kode Promo — Masukkan jika punya (opsional)
  6. Lanjut Pembayaran — Klik tombol untuk lanjut ke Step 2
  7. Scan QRIS — Bayar sesuai nominal yang tertera
  8. Klik "Saya Sudah Bayar" — Admin akan memverifikasi dalam 24 jam
Simpan bukti transfer sampai admin mengkonfirmasi. Konfirmasi akan dikirim via WhatsApp ke nomor yang didaftarkan.

Dashboard Admin

Akses dashboard di /dashboard.html dengan token admin.

Tab yang Tersedia

  • Kelola ID — Tambah, hapus, tandai terjual, upload massal
  • Pembayaran — Lihat semua pesanan, konfirmasi pembayaran
  • Promo — Buat & kelola kode diskon
  • Musik — Tambah, hapus, urutkan playlist
  • Banner — Kelola banner slider dari database
  • Popup — Atur konten popup promo
  • Edit File — Edit HTML langsung, push ke GitHub
  • Pengaturan — Harga tier, biaya admin
  • IP Banned & Broadcast — Kelola spam & kirim info ke semua user
  • Report — Balas pesan live chat dari user
  • Reset DB — Hapus data ID, payment, likes (banner/musik/popup aman)

Environment Variables

Set di Vercel Dashboard → Settings → Environment Variables:

# Wajib MONGODB_URI=mongodb+srv://user:pass@cluster.mongodb.net/lapakid ADMIN_TOKEN=your-secret-admin-token # Untuk GitHub File Editor GITHUB_TOKEN=ghp_xxxxxxxxxxxx # Repo sudah hardcode: syannnho/nataza (branch: main)
Jangan commit file .env ke GitHub. Gunakan Vercel Dashboard untuk set environment variables dengan aman.

Support

Butuh bantuan? Gunakan fitur Live Chat di halaman utama atau hubungi admin langsung.

Untuk laporan bug atau request fitur, gunakan fitur Report di halaman utama (icon CS di pojok kanan bawah).

Tips Umum

  • Pastikan MongoDB Atlas cluster berstatus Active
  • Whitelist IP 0.0.0.0/0 di MongoDB Network Access untuk Vercel
  • GitHub Token harus memiliki scope repo untuk Edit File berfungsi
  • Setelah push file via dashboard, Vercel akan auto-deploy dalam ~1 menit
